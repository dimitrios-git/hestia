# hestia theming testbench

Generated by the `testbench` Ansible role (`enable_testbench`). Everything here
is disposable ‚Äî delete the folder and re-run the role to regenerate it.

## code/ ‚Äî syntax-theme fixtures

Short samples per language, packed with the constructs that reveal theme
differences: comments (line/block/doc/TODO), strings + escapes + interpolation,
numbers (int/float/hex), booleans/null, keywords, symbolic vs word operators,
function definitions vs calls, builtins (functions/types/constants), parameters,
self/this, type annotations, decorators/attributes, member access, macros.

Open the same file across the tools you're theming and compare:

    vim  code/sample.py      # regex syntax
    nvim code/sample.py      # treesitter
    bat  code/sample.py      # TextMate (same family as VS Code / web Shiki)

VS Code and the web (Shiki) render from the same palette + scopes, so they
should match bat by role.

## files/ ‚Äî file-type (dircolors/ls) fixtures

One entry per `ls`/dircolors category, so `ls --color` / vifm / yazi / ranger
file-type colouring can be checked at a glance:

- extensions: archives, images, video, audio, docs, markdown, code
- directory; sticky+other-writable (`sticky-tmp`, like /tmp); sticky-only;
  other-writable; setgid dir
- executable, setuid, setgid files
- valid symlink, broken symlink (orphan), hardlink
- FIFO (named pipe), unix socket, empty file, dotfile, extension-less (Makefile)

Try:

    ls --color=always -la files/
    vifm files/

Block/char device nodes (bd/cd) need root (`mknod`) ‚Äî not created here; inspect
real ones under /dev if you need them.
